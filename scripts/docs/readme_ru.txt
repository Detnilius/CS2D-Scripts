#include
	include.load(modname, [...]) 
	-- Загружает указаный модуль. Если модуль загружен, то функция возвращает сохраненое
	-- значение, не загружая модуль заново. При загрузке модуля можно указать его параметры.
	-- Параметры можно получить внутри модуля с помощью синтаксиса "arg1, arg2 = ...". Первым
	-- аргументом всегда будет ссылка на таблицу 'include'.
	-- Можно использовать таблицу 'include' как функцию 'include.load', т.е. "include('mymodule')" эквивалентно "include.load('mymodule')".
	
	include.path([path1, ...])
	-- Добавляет указаные пути в таблицу путей поиска. Знак вопроса '?' заменяется текущим
	-- названием модуля, т.е. путь 'path/to/modules/?.lua' при загрузке модуля `include('mymodule')`
	-- будет 'path/to/modules/mymodule.lua'.
	-- Если аргументов нет, возвращает таблицу с путями.
	
	include.debug(modname, [enabled])
	-- Включает/выключает режим отладки для модуля. Если 'enabled' не указан, то возвращает
	-- отладочную таблицу модуля. Пример:
	--
	-- include.debug('mymodule', true) 
	-- moduleDebug = include.debug('mymodule')
	-- print(moduleDebug, moduleDebug.enabled)
	--
	-- Фактически, если модуль ещё не загружен, то мы всё равно может получить отладочную таблицу модуля.
	-- Получить отладочную таблицу в самом модуле можно так:
	--
	-- include = (...)
	-- moduleDebug = include.debug('mymodule')
	-- print(moduleDebug, moduleDebug.enabled)
	
	include.destroy(modname)
	-- Удаляет указаный модуль из загруженных. Если у модуля есть таблица, и в этой таблице есть метатаблица с
	-- функцией '__destroy' то он выполняет её. В ней можно сделать настройки для удаления модуля.

#timer
	timer.add([ms, func, count, ...])
	-- Более удобная обёртка для обычнго таймера CS2D. Один таймер как объект.
	-- Создаёт и возвращает объект таймера. Если параметры не указаны, создаёт таймер
	-- с стандартными значениями (по сути пустой таймер).
	-- Можно использовать таблицу 'timer' как функцию 'timer.add', т.е. "timer(...)" эквивалентно "timer.add(...)".
	-- Так-же если второй аргумент вызова таблицы 'timer' будет строка, то сработает стандартный таймер CS2D.
	-- Аргументы функции:
	-- 'ms' - Число, время в миллисекундах (интервал), через которое будет вызыватся 'func'. (Default '100')
	-- 'func' - Функция, которая будет вызываться по истечению времени 'ms', 'count' раз.
				Последним аргументом всегда, автоматически, будет вставляться
				сам таймер (ссылка на таблицу объекта таймера).
				Объект таймера не запустится (не унижчтожится сам), если параметр
				'func' не будет указан. (Default 'nil')
	-- 'count' - Число, количество повторений функции 'func'. Значение 0 или меньше будет
				бесконечно вызывать функцию. Используй метод obj:destroy() 
				чтобы удалить таймер, obj.state = 0 или
				obj.count = 1 чтобы остановить. (Default '0')
	-- '...' - Аргументы, которые будут переданы в функцию 'func'.
	--
	-- Методы объекта:
	-- obj:destroy() - Уничтожает таймер. После унижчтожения, доступ к свойствам
	--					таймера будет не возможен. Объект будет как пустая таблица.
	--
	-- Свойства объекта:
	-- obj._tm - (число, float) Время (возвращаемое функцией os.clock()), 
								когда произошел последний вызов функции таймера.
								Используется для отслеживания интервала и определения,
								когда должен произойти следующий вызов функции таймера.
	-- obj._address - (строка) Адрес таблицы объекта таймера в памяти. В основном
						используется при 'tostring(obj)', 'print(obj)' НЕ БОЛЕЕ.
	-- obj.interval - (число) Время в миллисекундах, через которое будет вызыватся
						функция таймера. Тоже самое 'ms'.
	-- obj.func - (функция) Функция, которая будет выполняться. Если функция не указана,
							таймер не будет выполнятся.
	-- obj.counter - (число) Количество повторений функции таймера. Когда это значение
							достигает 1, таймер останавливается 'obj.state = 0' или унижчтожается.
	-- obj.state - (число) Состояние таймера. (Больше 0 - активный, Меньше или 0 - неактивный)
	-- obj.iteration - (число) Количество вызовов функции таймера.
	-- obj.autodestroy - (boolean) Если установлено в 'true', таймер автоматически уничтожится
									после того, как значение 'counter' станет равным 1. (Default 'true')
	-- obj.arg - (таблица) - Массив аргументов, которые передаются в функцию таймера.
							Последний аргумент всегда будет сам 'obj', ссылка на
							таблицу объекта. (Для чтого чтобы управлять таймером в функции)
							
	Модуль имеет глобальную функцию, название функции в переменной 'GLOBAL_VARNAME', для того 
	чтобы таймер CS2D обрабатывал все остальные таймера. Удалив функцию, таймера перестанут работать.
	При каждом добавлении таймера, проверяется наличие данной функции, если её нет, то создается новая.

#hook
	hook.add(name, func, [par1, par2])
	-- Более удобная обёртка для обычнго хука CS2D. Создаёт и возвращает объект хука.
	-- Можно использовать таблицу 'hook' как функцию 'hook.add', т.е. "hook(...)" эквивалентно "hook.add(...)".
	-- 'name' - (строка) Название хука.
	-- 'func' - (функция) Функция, которая будет выполнена при срабатывании хука.
	-- 'par1' - (число/boolean, optional) Первый доп. параметр. Если 'Число' то задает приоритет выполнения хука.
	-- 'par2' - (число/boolean, optional) Второй доп. параметр. Если 'true' то хук будет пользовательским.
	--
	-- Методы объекта:
	-- obj:destroy() - Удаляет хук из списка активных хуков и выполняет очистку объекта.
	--
	-- Свойства объекта:
	-- obj._address - (строка) Адрес таблицы объекта хука в памяти. В основном
						используется при 'tostring(obj)', 'print(obj)' НЕ БОЛЕЕ.
	-- obj.name - (строка) Имя события или хука, к которому привязан данный объект.
							Это свойство определяет, к какому событию будет выполняться данная функция хука.
	-- obj.func - (функция) Функция, которая будет выполнена при срабатывании хука.
							Это свойство содержит ссылку на функцию, которая будет вызвана в момент срабатывания хука.
	-- obj.custom - (boolean) Флаг, указывающий, является ли данный хук пользовательским (custom) или нет.
								Если значение 'true', то это пользовательский хук (нужно самому контролировать
								вызов хуков). В противном случае хук будет рассматриваться как стандартный (игра сама вызывает хуки).
	-- obj.priority - (число) Приоритет хука. Числовое значение, определяющее порядок выполнения хуков при
								срабатывании события. Хуки с более высоким приоритетом будут выполняться
								позднее хуков с более низким приоритетом.
	-- obj.enabled - (boolean) Флаг, определяющий, активен ли данный хук. Если значение 'true', то хук активен и будет
								выполняться при срабатывании события. Если значение 'false', хук не будет выполняться.
	
	Модуль имеет глобальную функцию, название функции в переменной 'GLOBAL_VARNAME'.
	Удалив функцию, вызов функции объектов перестанут работать.
	Глобальную функцию можно использовать для вызова как обычных, так и 'пользовательских' хуков.
